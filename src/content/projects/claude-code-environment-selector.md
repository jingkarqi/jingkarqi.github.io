这是一个典型的开发者效率工具。它解决的问题不大，但非常具体：多模型供应商并存时，环境切换经常又慢又乱。

## 项目价值

- 通过配置文件统一供应商定义
- 把环境变量切换做成可重复动作
- 用别名和交互式选择降低日常使用成本

## 为什么会放在主展示里

因为它体现的是工程实用性。不是所有好项目都要“大”，但它们应该足够贴近真实工作流。
